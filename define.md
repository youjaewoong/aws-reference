
#### Amazon Redshift 클러스터
 - 데이터웨어하우스 솔루션
 - 집계쿼리 수행 가능 솔루션
 - 데이터를 영구 보관하고 거의 실시간쿼리 가능

#### Amazon EMR 클러스터 (이전 이름Amazon Elastic MapReduce)
 - 빅 데이터 프레임워크 실행을 간소화하여 방대한 양의 데이터를 병렬 처리하고 AWS 분석하는 관리형 클러스터 플랫폼입니다
 - ERM은 빅데이터 처리 솔루션

#### Elastic File System(Amazon EFS) 
 - 여러 가용영역의 EC2 인스턴스에서 동시 연결 가능한 파일 스토리지 솔루션
 - EFS Infrequent Access(EFS IA) : 메우 드믈게 엑세스 될때 사용되는 솔루션

#### EC2
 - 동일한 가용용역 배치 : 전송 비용을 줄이기 위한 솔루션
 - Systems Manager Run Command 를 사용하여 타사 소프트웨어를 즉시 패치 할수 있습니다.
 - 클러스터 배치 그룹
   - 서버를 고속 네트워크로 연결로 그룹화하여 네티워크 처리량 최대화 할수 있습니다.
   - 대기시간이 짧은 네트워크 성능과 높은 네트워크 처리량을 처리 할수 있습니다.
 - 전용호스트
   - 하드웨어를 공유하지 않음
   - 고객 전용의 EC2 인스턴스 용량을 갖춘 물리적 서버입니다.
   - 전용 호스트를 사용하면 Windows Server, Microsoft SQL Server, SUSE 및 Linux Enterprise Server를 포함한 기존 소켓당, 코어당 또는 VM당 소프트웨어 라이선스를 사용할 수 있습니다
 - 최대 절전모드 : 인스턴스를 시작할 때 저장된 인메모리 상태가 복원되어 빠르게 시작됩니다.
 
#### IAM
 - 암호 만료 기간 설정에 따른 사용자 처리
 - 권한 경계 설정 : 관리자 정책 연결을 명시적으로 거부할 경우
 - 신뢰 정책 : dev -> prod 계정위임 처리

#### DynamDB
 - 트래픽 양에 따라 확장
 - 지정 시간복구 (point-in-time recovery) 구성
 - RPO 복구의 경우 원하는 시점으로 복원
 - 높은 읽기 처리량지원, 서버리스 서비스로 운영 오버헤드 최소화 하는 DB
 - 마이크로 초 단위 응답시간 제공, 개발 및 운영오버헤드 최소화
 - Accelerator(DAX) : 캐싱기능을 이용해 읽기 성능 향상. 운영 오버헤드가 없고 애플리케이션 수정필요없음
 - 스트림 활성화 기능
 - 읽기/쓰기 용량모드
   - 온디맨드용량모드 : 읽기/쓰기 처리량을 자동으로 조정, 트래픽 예측이 불가능한 경우 사용
   - 프로비저닝 모드 : RCU(읽기 용량 단위) 및 WCK(쓰기 용량 단위) 지정, 트래픽이 예측 가능할 경우 사용
   
#### Amazon ElastiCache
 - 가장낮은 대기 시간 제공
 - 연결을 끊었다가 다시 연결해도 세션 데이터 유지

#### KMS (SSE-KMS)
 - 키를 매년 자동 순환 시키는 옵션
 - AWS가 마스터키 관리, 사용자가 데이터키 관리, 자동 키관리 및 교체, 키에 엑세스할 수 있는 사람 제어
 - Lambda 환경변수를 암호화 처리하는 경우
   - Lambda 실행역할에 권한 부여
   - KMS 키 정잭에서 Lambda 실행 역할 허용
 - 다중리전키 : 여러 리전에서 동일한 키를 가지고 있는 것처럼 서로 바꿔서 사용할 수 있는 다른 AWS 리전의 AWS KMS keys입니다.

#### CMK (고객 마스터키)
 - 키 사용자 엑세스 제어 가능

#### NLB (Network Load Balacner)
 - 고정 IP 제공
 - TCP 트래픽 처리
  - 가용성 및 확장성 제공
 - TCP, UDP 트래픽을 처리하며 초당 수백만의 대용량 트래픽을 처리합니다.

#### ALB (Application Load Balancer)
 - HTTP 트래픽 처리
 - 세션 선호도
   - 활성화시 동일 EC2인스턴스에 대한 요청을 유지하는 기능
   - 비활성화 시 대상그룹의 EC2 분산

#### GLB (Gateway Load Balacner)
 - 방화벽, 침입탐지 시스템 같은 가상 어플라이언스를  배포하여 트래픽 검사 할수 있습니다.

#### S3
 - Transfer Acceleration : 전송 가속화 솔루션 (비용증가)
 - 불완전한 멀티파트 업로드를 삭제하는 S3 수명 주기 정책 활성화 (비용감소)
 - Intelligent-Tiering 스토리지 사용
   - 수명주기 설정
   - 액세스 패턴을 알 수 없거나 예측할 수 없는, 또는 액세스 패턴이 변화하는 데이터의 경우 S3 Intelligent-Tiering이 이상적인 스토리지 클래스입니다. 
 - Standard :
   - 페타바이트 규모의 데이터 저장 가능
   - 즉시검색 기능
   - 최소 과금 기간없음
- Standard-IA : 비용효율적인 저장 솔루션 (엑세스 빈도가 낮은 파일 처리에 효율적)
   - 최소 과금 기간 30일
 - Glacier
   - 아카이브 검색 옵션
     - Expedited(긴급) : 1~ 5분 검색
     - Standard(표준) : 3-5시간
     - Bulk(대량) : 5~12시간
   - 최소 과금 기간 90일
 - Glacier Deppe Archive
   - 최소 과금 기간 180일
 - 우발적인 삭제 방지 : 버전관리, MFA 삭제 활성화
 - 미리 서명된 URL을 생성하여 제한된 시간동안 엑세스를 제한할수 있습니다.
 - S3 버킷 공유 : 교차 계정 엑세스 사용하여 S3 버킷 정책 설정
 - OAI(Origin Acces Identitiy) : OAI를 CloudFront 배포에 할당 OAI만 읽기 권한을 갖도록 S3 버킷권한을 구성합니다.
 - 지능형 계층화 : 엑세스 패턴을 알 수 없는 경우 적합
 - 스토리즈 중 가격이 제일 저렴합니다.
 - Object Lambda : 데이터를 반환하기전에 반환 데이터를 변환 할수 있습니다.
 - 인벤토리 : 객체를 감사하는 기능
 - Storage Lens ; 버킷의 엑세스 패턴을 분석합니다.

### Athena
 - 온디맨드 보고서 및 쿼리 실행 ( S3의 데이터를 쿼리할 수있는 서비스)
   
#### API Gateway
 - 확장 가능한 탄력적인 API 솔루션
 
#### Storage Gateway 파일 게이트웨이
 - 온프로미스와 AWS 스토리지 연결
 - iSCSi  가상 테이프 라이브러리 (VTL) : 백업 어플리케이션 연결
   - NAS : 한 원격 스토리지 볼륨을 다수의 사용자가 IP를 통해 접속.
   - iSCSI : 드라이브 마운트인데, 물리적인 연결이 아니라 1:1 매치(원격 드라이브 마운트)
 - 캐시볼륨 : 최근 엑세스한 데이터만 저장하는 캐시 볼륨

#### AWS DataSync
 - 데이터 온라인 마이그레이션 솔루션
 - 온프로미스 nfs 서버에서 s3 버킷으로 마이그레이션 처리
 - 기존 서비스를 중단하지 않고 EFS로 마이그레이션 할 수 있습니다.
 - 데이터 무결성 기능이 있습니다.

#### Snowball 엣지 스토리지
 - 오프라인 마이그레이션 솔루션

#### kinesis Data Streams
 - 모든 규모의 데이터 스트림을 쉽게 캡처, 처리 및 저장할 수 있는 서버리스 스트리밍 데이터 서비스입니다.
 - 365일까지 데이터 보관 가능

#### kinesis Data Analytics
 - 실시간 스트리밍 처리 솔루션
 - 표준 SQL을 이용하여 스트리밍 데이터를 처리하고 분석할 수 있습니다.
 - 스트리밍 소스에 대해 강력한 SQL 코드를 작성하고 실행하여 시계열 분석을 수행하고, 실시간 대시보드를 제공하고, 실시간 지표를 생성할 수 있습니다.

#### Kinesis Data Firehose
 - 스트리밍 데이터를 안정적으로 캡처하고 전환하여 데이터 레이크, 데이터 스토어, 분석 서비스에 전달하는 추출, 전환, 적재(ETL) 서비스입니다.
 - privateLink 를 사용하여 VPC에서 Kinesis Data Firehose용 인터페이스 VPC 엔드포인트 생성
 - 온프로미스 네트워크와 AWS 간 1Gbps AWS Direct Connect 연결 설정
 - PrivateLink 엔드포인트를 사용하여 온프로미스에서 Kinesis Data Firehose로 데이터 전송
 - 서로다른 AWS 게정에서 VPC 엔드포인트를 통한 PrivateLink를 사용하여 비공개 연결이 가능합니다.

#### SQS (Simple Queue Service)
 - 대기열 프로비저닝
 - 대기열 폴링 ec2 인스턴스 구성
 - 인스턴스 당 백로그 계산을 기반으로 메트릭 생성, 메트릭을 기반으로 AutoScailng 그룹을 확장합니다.
 - 자신의 권한을 포기하지 않고 다른 회사에 SQS 엑세스 제공
 - 가시성 제한 시간(visibility timeout)
   - 다른 소비자가 메시지를 중복 처리하지 못하도록 SQS는 가시성 제한 시간을 설정합니다.
   - SQS는 다른 소비자가 메시지를 수신하고 처리하지 못 하도록 하는 기간입니다.
   - 메시지의 기본 표시 제한 시간은 30초 입니다. 최소값은 0초 입니다. 최대 값은 12시간입니다.

#### FSx (Windows 기반 파일시스템)
 - Windows 인스턴스 내 파일 시스템 공유
 - 고성능 컴퓨팅(HPC) 환경에 Lustre 영구(지속적) 파일 시스템 사용 가능

#### Elastic Beanstalk
 - 애플리케이션을 실행하는 인프라에 대해 자세히 알지 못해도 AWS 클라우드에서 애플리케이션을 신속하게 배포하고 관리할 수 있습니다. 
 - 다중 AZ 로드 발란싱

#### ACM (AWS Certificate Manager)
 - HTTPS 인증서 (SSL/TLS)
 - 외부 인증서를 처리할 경우 수동으로 교체해야 합니다.
 - ACM의 SSL 인증서를 사용하는 HTTPS 리스너로 Application Load Balancer 생성 하면 EC2의 용량을 사용하지 않는다.

#### CloudFront
 - 배포생성 및 가격 등급 선택
 - 요청 증가 시 엣지로케이션에 콘텐츠 캐싱으로 트래픽 분산
 - 엣지로케이션에서 데이터를 압축하여 데이터 전송비용 감소
 - 국가별 엑세스 거부 설정 가능
 - 필드 수준 암호화 프로필 구성 : 추가적인 암호화 레벨 제공, 엣지로케이션에서 비대칭 암호화 키를 사용하여 데이터를 추가로 암호화 합니다.

#### VCP (Virtual Private Cloud)
 - 정의한 논리적으로 격리된 가상 네트워크에서 AWS 리소스를 시작할 수 있습니다.
 - 네트워크는 AWS의 확장 가능한 인프라를 사용한다는 이점과 함께 고객의 자체 데이터 센터에서 운영하는 기존 네트워크와 매우 유사합니다.
 - VPC 엔드포인트
   - 데이터 출력 비용을 줄일수 있습니다.
   - DynamoDB, S3서비스를 프라이빗하게 연결할 수 있습니다. (인터넷을 통과하지 않아도됨)
 - 라우팅 테이블 구성에 따라 인터넷 엑세스 가능
 - 피어링 : VPC-A, VPC-B 간에 프라이빗 연결이기에 보안 엑세스에 대한 요구사항 만족, 단일 지점 장애가 없으며 대역폭 제한도 없음

#### AMI 
 - 인스턴스를 시작하는 데 필요한 정보를 제공하는 AWS에서 지원되고 유지 관리되는 이미지입니다.
 - EC2 인스턴스 복사 및 다른 리전 지정이 가능합니다.

#### Auto Scaling
 - 대상 추적 조정 정책 : cpu 사용률에 따라 Auto Scaling 그룹을 조정합니다.
 - 예정된 스케일링을 사용하여 최소, 최대 용량을 0 으로 변경하면 인스턴스가 작동하지 않습니다.
 - 예측 조정 활성화 : 머신러닝을 사용하여 CloudWath의 기록 데이터를 기반으로 필요량을 예측
 - 동적 스케일링 : 임의의 요일에 갑작스러운 증가에 스케일링이 가능
 - 수명주기 후크 : 인스턴스 시작 및 종료 시 이벤트 트리거를 할수 있습니다.
 - warm pools : 인스턴스를 웜풀에 추가하면 auto scaling 확장시 인스턴스를 빠르게 시작 합니다.

#### 온디맨드
 - 장기 약정 없이 컴퓨팅 파워에 대해 시간당 또는 초당(최소 60초) 비용을 지불하게 됩니다
 - 단기 인스턴스 사용에 적합합니다.

#### Savings Plan
 - EC2 instance Savings Plan : EC2에 대한 할인 모델
 - Compute Savings Plan : EC2, Lambda 할인 모델
 - 1년 또는 3년 동안의 컴퓨팅 성능 사용을 약정하는 대가로 할인 혜택을 제공받을 수 있는 할인 모델입니다.
 - 인스턴스 패밀리와 타입 모두 변경 가능

#### CloudWath
 - 머신러닝을 사용하여 올바른 스케일링 정책 데이터기반의 용량 필요량을 예측할수 있습니다.
 - 하드웨어 장애가 발생 할 경우 경보를 통한 손상된 하드웨어를 복수 할수 있습니다.

#### 전용 호스트
 - 하드웨어를 공유하지 않음

#### NAT 게이트웨이
 - PRIVATE IP에서 인터넷으로 요청을 보낼때 해당 요청이 Nat을 제공하는 라우터를 통과하게되면
   NAT 라우터는 주소 변환 테이블에 가지고있던 PrivateIP를 PublicIP로 IP를 변환하여 요청을 보내고 변환 내용을 Nat변환 테이블에 기록합니다.
 - 많은 트래픽 지원 및  고가용성, 내결함성, 자동 확정성을 충족합니다.
 - 퍼블릭 서브넷에 배포, 프라이빗 서브넷의 EC2  인터넷과 연결하기 위해 필요
 - 웹 계층 서브넷에 대한 네트워크 ACL 수정하여 IP주소에 대한 인바운드 규칙을 추가할 수 있습니다.
 - IPv4 주소만 라우팅 가능, IPV6 는 송신 전용 인터넷 게이트웨이로 가능 

#### AWS Fargate (컨테이너)
 - Amazon EC2 인스턴스의 서버나 클러스터를 관리할 필요 없이 컨테이너를 실행하기 위해 Amazon ECS에 사용할 수 있는 기술입니다
 - 가용성이 높고 수요 증가에 맞게 확장 가능
 - 트랙픽 증가에 따라 Auto Scaling 구성 할 수 있습니다.

#### Global Accelerator
 - 2개의 고정 IP를 가짐
 - 글로벌 사용자에게 최적화된 경로 제공으로 라우팅 속도를 빠르게 해줍니다.
 - 여러 리전에 배포된 ALB 중에 트래픽을 정상 엔드포인트로 리디렉션 가능

#### Secrets Manager
 - 자격증명 자동 교체 가능, 자격증명 관리 서비스

#### Direct Connect
 - 온프로미스와 AWS 전용선 연결 (최대성능을 만족할수 있습니다.)
 - 암호화 되지 않은 채널 전송 방법
 - 프라이빗 가상 인터페이스(VIF) : Direct connect 연결을 시작하려면 선택 사항 입니다.

#### Aurora
 - 복제본 추가 시 트래픽이 분산되어 요청속도가 빨라집니다.
 - 단일 DB 인스턴스로 구성되어도 데이터는 여러 가용영역에 복제 및 분산 저정 됨
 - 글로벌 데이터베이스 클러스터 : 다른 리전으로 데이터베이스를 복제하는 기능
 - Serverless : 사용자가 관리할 필요없는 자동화 솔루션
 - AutoScaling : 읽기 로드를 자동으로 조정
 - 지속적인 복제 작업
    - 마이그레이션 복제 서버 생성
    - 지속적인 변경 사항을 동기화 하면서 복제
 - 프록시를 통해 연결 수 제한으로 연결 시간 초과 문제를 해결할 수 있습니다.
 - cloning : 프로덕션 DB 클러스터에 영향없이 Staigng DB 생성 가능

#### VPN
 - 암호화된 채널 전송
 - 1Gbps 인터넷으로 10TB 전송에 20시간 소요

#### Inspector
 - EC2 및 컨테이너 워크로드 소프트웨어 취약점 관리 서비스

#### WAF
 - DDoS 공격 방어 가능
 - 특정 국가에서만 엑세스 제한 가능
 - XSS, SQL Injection 공격으로부터 보호 가능

#### Shield Advanced
 - DDos 공격 방지 솔루션

#### Security Hub
  - 보안 모범 사례를 확인하고, 경고를 집계하고, 자동화된 수정을 활성화하는 클라우드 보안 태세관리(CSPM) 서비스 입니다.

#### Macie
 - 개인식별정보 같은 민감한 데이터를 검색하고 보호하는 서비스

#### GuardDuty
 - AWS 계정 및 워크로드에서 악의적 활동을 모니터링하고 상세한 보안 결과를 제공하는 지능형 위협 탐지 서비스

#### 네트워크 ACL
 - 네트워크레벨 방화벽
 - 서브넷 수준에서 특정 인바운드 또는 아웃바운드 트래픽을 허용하거나 거부합니다.
 - VPC에 대한 기본 네트워크 ACL을 사용하거나 보안 그룹에 대한 규칙과 유사한 규칙을 사용하여 VPC에 대한 사용자 지정 네트워크 ACL을 생성하여 VPC에 보안 계층을 추가할 수 있습니다.
 - 비용이 발생하지 않습니다.

#### Cognito
 - 애플리케이션에 대한 로그인 및 인증 제공 기능
 - 사용자 풀 권한부여자를 구성하면 REST API 에 대한 엑세스 제어가 가능합니다.

#### SAML (Security Assertion Markup Language) 2.0 기반 페데레이션
 - 온프로미스 AD 사용자를 AWS 액세스와 통합

#### SCP(서비스 제어 정책)
 - 조직의 계정에 대한 사용가능한 권한을 제어할수 있습니다.

#### ECR (Elastic Container Registory)
 - Docker 이미지를 저장하고 관리하는 솔루션 입니다.

#### Transit Gateway
 - 모든 VPC, VPN을 하나의 Transit Gateway로 연결할 수 있습니다.
 - VPC 피어링보다 가격이 비쌉니다

#### AWS Backup
 - 최소한의 노력으로 백업 및 복구 하는 솔루션

#### EC2 인스턴스 스토어
 - 가장 고성능 임시 블록 스토리지
 - 초당 수백만 개의 트랜잭션을 지원

#### 스냅샷
 - 데이터베이스와 스냅샵이 항상 암호화 처리 : 최신 DB 스냅샵 사본을 암호화, 암호화된 스냅샷을 복원하여 기존 DB 인스턴스 교체

#### FSx for Lustre
 - 머신러닝, 빅데이터를 위한 고성능 스토리지 솔루션
 - HPC 워크로드를 처리하는 고성능 병렬처리 파일 스토리지

#### Route 53
 - 장애조치라우팅 정책 : 상태확인을 통해 장애 발생시 다른 리전의 ALB로 페일오버 가능
 - 쿼리 로깅 : DNS 응답을 로깅하는 기능
 - 라우팅 정책
   - 장애조치 정책 : 기본라우팅이 실패하면 보조로 자동 라우팅 되는 방식
   - 가중치 정책 : 가중치 기준으로 라우팅
   - 다중값 응답 라우팅 정책 : DNS레코드에 있는 다수의 값을 무작위로 반환
   - 지리적 위치 : 가까운곳으로 라우팅

#### Lambda
 - API 설계 후 요청한 만큼 비용 지불을 할수 있습니다.
 - 최대 실행 시간 15분
 - 예약된조정 : 프로비저닝된 동시성은 함수에 할당하려는 사전 초기화된 실행 환경의 수입니다.

#### Transfer for SFTP
 - SFTP를 사용하여 파일을 교환하는 솔루션

#### Glue
 - 완전 관리형 ETL(추출, 변환 및 로드) 서비스로, 간단하게 여러 데이터 스토어 및 스트림 간에 원하는 데이터를 분류, 정리, 보강, 이동합니다.
 - ETL: 추출(Extract), 변환(Transform), 로드(Load) 처리하는 솔루션 입니다.

#### EventBridge (CloudWatch Events)
 - 애플리케이션 구성 요소를 서로 연결하는 서버리스 서비스로, 확장 가능한 이벤트 기반 애플리케이션을 더 쉽게 구축할 수 있습니다
 - 이벤트를 자동 전송하는 서비스

#### CloudWatch
 - 복합적인 경보를 설정할 수 있습니다. ( cpu 사용률, 디스크 IOS읽기)
 - AWS 계정이없는 관리자에게 이메일 공유 가능

#### Amplify
 - Amplify는 AWS에서 풀 스택 웹 및 모바일 앱을 구축하는 데 필요한 모든 것을 제공합니다.
 - 모바일 및 웹애플리케이션 개발 도구

#### SSM (Systems Session Manager)
 - EC2 인스턴스, 엣지 디바이스, 온프레미스 서버 및 가상 머신을 브라우저 기반 셸 또는 AWS CLI를 통해 관리할 수 있습니다.
 - 최소한의 운영 오버해드 충족합니다.

#### 재해복구전략
 - 파일럿 라이트(Pilot light)
   - 먼저 추가 조치를 취하지 않으면 요청을 처리할 수 없습니다.
   - 코어 시스템만 복제 합니다.
 - 웜 스탠바이(Warm Standby)
   - 트래픽을 (감소된 용량 수준에서) 즉시 처리할 수 있습니다.
   - 전체 시스,템을 작은 스케일로 구성하여 복제 합니다.

#### Rekognition
 - 이미지 및 비디오에서 부적절하거나 원치 않거나 불퀘감을 주는 콘텐츠 탐지 가능

#### Comprehend
 - 텍스트 안에서 특정 항목을 찾아내는 머신러닝 서비스

#### SageMaker
 - 머신러닝 모델을 구축, 훈련 및 배포하는 서비스

#### Organizations
 - 생성한 여러 AWS 계정을 조직에 통합하고 중앙에서 관리할 수 있는 계정 관리 서비스입니다.
 - AWS Organizations의 계정 관리 및 통합 결제 기능을 활용하면 기업의 예산, 보안 및 규정 준수 요구 사항을 보다 잘 충족할 수 있습니다.
 - 조직의 관리자로서 조직에서 계정을 생성하고 기존 계정을 조직에 초대할 수 있습니다.

#### QuickSight
 - 데이터 시각화를 제공하는 서비스 입니다.
 - 사용자 및 그룹에 공유가 가능합니다.

#### AWS 스팟플릿
 - 스팟 집합을 통한 온디멘드 용량을 비용효율적으로 충족할수 있습니다.

#### SES (Simple Email Service)
 - 이메일을 보내는 서비스
 - 메시지 폼을 구성하여 서버측에 전달이 가능합니다.

#### EBS (Elastic Block Store)
 - 암호화된 볼륨을 생성함으로 써 데이터를 암호화 처리 할 수 있습니다.
 - GP2 볼륨보다 GP3 볼륨이 더 저렴하다.

#### RDS
 - 스토리지 유형
   - 범용 SSD (gp2 및 gp3)
   - 프로비저닝된 IOPS SSD(io1)
   - 마그네틱(표준)

#### Cost Explorer
 - 시간에 따른 AWS 비용과 사용량을 시각화, 이해 및 관리할 수 있는 손쉬운 인터페이스를 제공합니다.
 - 비용 보고서는 임계값을 초과하는 즉시 알람을 받을수 없습니다.

#### Transfer Family
 - FTP 클라이언트에 대한 최소한의 변경으로 수신 데이터 파일을 처리할 수 있는 aws 솔루션입니다.
   S3 또는 EFS 를 스토리지로 지정 가능합니다.

#### EKS
 - 포드에 IAM 역할을 연결 할수 있습니다.
 - VPC 엔드포인트를 생성하여 프라이빗한 연결을 할수 있습니다.
 - 커넥터 : 외부(온프레미스) kubernets 클러스터를 Amazon EKS에 등록하여 관리 가능

#### Control Tower
 - 다중계정 환경을 쉽게 관리하는 서비스
 - OU 변경에 대한 dRIFT 감지에 대해 알림 설정이 가능합니다.

#### 인스턴스 스케줄러
 - EC2와 RDS를 일정에 따라 중지 및 시작 구성이 가능합니다.

#### Outposts
 -  인프라, 서비스, API 및 도구를 고객 온프레미스로 확장하는 완전관리형 서비스입니다.

#### Lake Formation
 - 분석 및 기계 학습을 위한 데이터를 중앙에서 관리하고, 보호하고, 전 세계적으로 공유할 수 있도록 지원합니다

#### 행수준 보안과 셀 수준 보안
 - 행 수준, 셀 수준 보안을 사용하면 민감한 데이터에 대한 액세스를 세부적으로 제어할 수 있습니다.

#### DMS (Database Migration Service)
 - 관계형 데이터베이스, 데이터 웨어하우스, NoSQL 데이터베이스 및 기타 유형의 데이터 스토어를 마이그레이션할 수 있는 클라우드 서비스입니다.

#### 게이트웨이 로드밸러서
 - 방화벽, 침입 탐지 및 방지 시스템, 심층 패킷 검사 시스템과 같은 가상 어플라이언스를 배포, 규모 조정 및 관리할 수 있습니다.

#### Amazon Elastic Transcoder
Amazon S3 Simple Storage Service (Amazon S3) 에 저장한 미디어 파일을 소비자 재생 디바이스에 필요한 형식의 미디어 파일로 변환할 수 있습니다.
예를 들어 대용량 고품질 미디어 파일을 사용자가 모바일 장치, 태블릿, 웹 브라우저 및 연결된 TV에서 재생할 수 있는 형식으로 변환할 수 있습니다.

#### AWS Trusted Advisor
클라우드에서 대규모로 운영하고, 보안 및 복원력을 개선하고, 성능을 높이고, 비용을 최적화할 수 있습니다.
Trusted Advisor는 비용 최적화, 성능, 복원력, 보안, 운영 우수성, 서비스 한도의 범주에서 모범 사례 검사를 사용하여 AWS 환경을 지속적으로 평가하고, 모범 사례와의 편차를 해결하기 위한 조치를 권장합니다
